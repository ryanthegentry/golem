> **Known issue (fixed Feb 26):** The VTXO listener previously used an AsyncIterable pattern
> that didn't match the SDK's callback-based `notifyIncomingFunds()` API. If testing an older
> build, the OOR send succeeds but the gateway can't detect the incoming VTXO. Update to the
> latest code.

## 13. Read-Only Wallet (Pubkey Only)

Quick validation that read-only mode works. First derive the public key, then query with it:

```bash
# Derive compressed pubkey from private key
npx tsx -e "
import * as secp from '@noble/secp256k1';
const privkey = Buffer.from('fixture', 'hex');
console.log(Buffer.from(secp.getPublicKey(privkey, true)).toString('hex'));
"
# Copy the output (03a294...)
```

Create `scripts/readonly-test.ts`:
```typescript
import { EventSource } from 'eventsource';
Object.assign(globalThis, { EventSource });

import { ReadonlyWallet, ReadonlySingleKey } from '@arkade-os/sdk';

async function main() {
  const pubkey = new Uint8Array(Buffer.from('<paste hex pubkey here>', 'hex'));
  const identity = new ReadonlySingleKey(pubkey);  // raw 33-byte Uint8Array, not {pubkey}
  const wallet = await ReadonlyWallet.create({      // .create() factory, not new + connect()
    network: 'mutinynet',
    identity,
    arkServerUrl: 'https://mutinynet.arkade.sh'
  });
  const balance = await wallet.getBalance();
  console.log('Read-only balance:', JSON.stringify(balance, null, 2));
  process.exit(0);
}
main();
```

```bash
npx tsx scripts/readonly-test.ts
```

Verify: returns correct balance matching `golem balance`, with zero key material used.

> **API notes:** `ReadonlySingleKey` takes a raw 33-byte `Uint8Array` as the constructor arg (not an object). `ReadonlyWallet` uses a static `.create()` factory (not `new` + `.connect()`).

## 14. Refresh Agent

```bash
GOLEM_SIGNER_KEY=fixture npm start
```

The server starts the refresh agent automatically. Check logs for:
- `RefreshAgent: polling every 60s`
- VTXO expiry checks
- Consolidation skip logic (if only 1 VTXO)

> On mutinynet with fresh VTXOs, no refresh will trigger — they're not near expiry. To validate the logic runs, just confirm the polling starts and the expiry check completes without error.

## Troubleshooting

**Faucet slow / block not mined:**
Wait longer, or use Ryan's pre-funded key:
```bash
npm run golem -- init --force
# Edit ~/.golem/config.json:
#   "privateKey": "fixture"
npm run golem -- balance
```

**Boltz "onchain coins could not be sent":**
Boltz has no ARK liquidity. Prime with submarine swap:
```bash
VOLTAGE_MACAROON="$VOLTAGE_MACAROON" npx tsx src/l402/test-live-manual.ts
```

**Voltage LND down:**
Check https://nodes.voltage.cloud or wait and retry. Nothing works without LND for the Lightning path.

**Balance doesn't reflect payment:**
Reverse swap is preconfirmed but not settled. Wait for the next Ark round (~1 min on mutinynet).
