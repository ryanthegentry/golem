# Spec 08: L402 Module

## Purpose
HTTP payment gateway implementing L402 protocol with V2 binary macaroons, dual-mode payment (Lightning + Ark OOR), response caching with dynamic pricing, rate limiting, and Boltz circuit breaker.

**This is the largest module — 132 tests in TS.**

## Sub-modules

1. **Macaroon** — HMAC-SHA256 mint/verify/parse (zero-dep)
2. **MacaroonStore** — SQLite anti-replay tracker
3. **InvoiceLimiter** — Rate limit pending unpaid invoices
4. **ResponseCache** — SQLite response cache with TTL
5. **Proxy** — HTTP reverse proxy to upstream
6. **Gateway** — L402 orchestrator (middleware)
7. **GatewayServer** — HTTP server setup (axum)
8. **InternalApi** — REST endpoints for 402index integration

## 1. Macaroon

### V2 Binary Identifier (66 bytes)

```
Offset  Len  Field           Details
0       2    version         uint16 BE, always 0
2       32   payment_hash    SHA-256(preimage)
34      4    root_key_id     Unique per-macaroon
38      28   padding         Zeros (reserved)
```

### API


### HMAC-SHA256 Chain (Zero-Dep Implementation)

```
root_key = random_32_bytes()
sig_0 = HMAC-SHA256(root_key, identifier)
sig_1 = HMAC-SHA256(sig_0, caveat_1)  // "time-before 2026-03-18T14:46:37Z"
sig_2 = HMAC-SHA256(sig_1, caveat_2)  // "expires_at = 1742310397"
...
sig_n = final signature (32 bytes, stored in V2 TLV)
```

### Verification Flow
1. Decode base64 → V2 binary
2. Extract identifier → payment_hash + root_key_id
3. Look up root key → error if not found
4. Replay HMAC chain from root_key through all caveats
5. Compare computed signature vs stored signature (constant-time)
6. Check `time-before` caveat: `now <= expiry`
7. Check `expires_at` caveat: `now < timestamp`
8. Constant-time preimage check: `SHA256(preimage) == payment_hash`

### Interop Fixture

File: `test-fixtures/golem-macaroon-interop.json`
- Contains: macaroon_base64, payment_hash, root_key_id, preimage, location, identifier_layout
- Validated against Go V2 binary format (Aperture compatible)
- **Rust implementation MUST produce identical binary output for same inputs**

## 2. MacaroonStore

```rust
pub struct MacaroonStore {
    pool: SqlitePool,
}

impl MacaroonStore {
    pub async fn new(pool: SqlitePool) -> Result<Self, Error>;
    pub async fn register(&self, payment_hash: &str, expires_at: u64, price_sats: u64) -> Result<(), Error>;
    pub async fn verify(&self, payment_hash: &str) -> (bool, u64);  // (valid, expires_at)
    pub async fn active_count(&self) -> u64;
    pub async fn cleanup(&self) -> u64;  // Deletes expired > 7 days
    pub async fn get(&self, payment_hash: &str) -> Option<MacaroonRecord>;
}
```

**Schema:** `active_macaroons(payment_hash TEXT PK, expires_at INT, price_sats INT, created_at INT, last_verified_at INT)`
**Behavior:** `register()` is INSERT OR IGNORE (idempotent). `verify()` returns false for unknown or expired. `cleanup()` deletes entries expired > 7 days.

**Cache key:** `SHA-256(url || method || body)` → 64-char hex.

## 5. Gateway Middleware

### Configuration

```rust
pub struct GatewayConfig {

    pub total_requests: u64,
    pub paid_requests: u64,
    pub challenges_issued: u64,
    pub total_sats_earned: u64,
    pub rate_limited: u64,
    pub lightning_paid_requests: u64,
    pub lightning_earned: u64,
    pub ark_paid_requests: u64,

    pub ark_pending_payments: u64,
    pub cache_hits: u64,
    pub cache_misses: u64,
    pub cache_sats_earned: u64,
}
```

### Flow (axum middleware)
1. Free paths → pass through
2. Cache lookup → set effective price (reduced if cache hit)
3. Check Authorization header → verify L402 token
4. If valid: track stats, post-auth method validation (405 if mismatch), serve cached or proxy
5. Rate limit check (per-IP, 30/min/IP)
6. Boltz circuit breaker check (503 if open)
7. Create Lightning invoice (SDK `get_ln_invoice`)
8. Optionally include Ark OOR payment option
9. Return 402 with WWW-Authenticate + JSON body

### Dual-Mode Payment (Ark OOR)
- 402 response includes `ark_payment: { address, amount, payment_id, macaroon }`
- Amount = effective_price + random(1-9999) sats (disambiguation)
- Gateway generates random preimage, stores in pending map
- Client polls `GET /l402/preimage?payment_id=X`
- VTXO detection via `subscribe_to_scripts()` gRPC stream
- On match: mark pending fulfilled, return preimage on next poll

### Boltz Circuit Breaker
- 5 failures in 60s window → open for 30s
- While open: return 503 immediately
- Reset on successful invoice creation

### Rate Limiter
- Sliding-window per-IP, 60s window
- Exempt: `/l402/preimage`
- Trust x-forwarded-for only if GOLEM_TRUSTED_PROXY set

## Test Specifications (from TS: 132 tests across 8 files)

### Security (security.test.ts)
- Constant-time preimage: wrong rejected, correct accepted, truncated rejected
- Replay protection: time-before caveat minted, macaroon expires after TTL
- Root key isolation: each macaroon unique key, cross-store fails, delete invalidates
- V2 binary: 66-byte identifier, version=0, payment_hash matches, padding zeros, round-trip
- Preimage: SHA256 manual verification, wrong hash rejected, empty rejected, tampered binary rejected
- L402 protocol: header format, parse L402/LSAT, reject non-L402, malformed, colon in preimage
- E2E: full mint→serialize→parse→verify cycle, caveats preserved
- FileRootKeyStore: persist, permissions 0600, delete persists, corrupt recovery
- Ark payment: 402 includes ark_payment, amount range, preimage endpoint (400/404/202/200)
- Cache-and-resell: reduced price, X-Golem-Cache header, streaming not cached, TTL race
- Boltz resilience: 503 on failure, retry success, circuit breaker 5→open→503→cooldown

### Response Cache (10 tests)
- Deterministic key, put/get, expired returns null, hit count, overwrite, FIFO eviction, prune, stats, earnings, binary data, persistence

### Internal API (12 tests)
- /l402/challenge: valid response, camelCase+snake_case+legacy, registered in store, invalid→400
- /l402/verify: valid→true, invalid→false, expired→false, missing→400
- /l402/status: all fields present, healthy=true, no auth required

### Interop (5 tests)

### Method Validation (9 tests)

### MacaroonStore (9 tests)

### Proxy (9 tests)
